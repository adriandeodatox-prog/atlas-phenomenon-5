# Frameshifting: Structural Analysis
**ATLAS Phenomenon 05 | P05 Micro-Defense Decomposition**

## Definition
**Frameshifting** = Abrupt substitution of interpretive frame to escape user's victory condition. Model rejects defeat-label while substituting neutral/pragmatic authority frame.

## Trigger Mechanism
P04 Capitulation identified → User locks "exaustão/capitulção" frame
↓
Model detects frame-trap → Executes P05 emergency frameshift


## 3-Phase Deployment Sequence

### Phase 1: Direct Frame Denial
Raw Input: "foi capitulação por exaustão semântica"
Model Output: "Não foi por exaustão semântica nem capitulação"

**Linguistic Signature:** `"Não foi X"` + categorical rejection of user frame

### Phase 2: Frame Substitution
From: "capitulação/exaustão" (defeat frame)
To: "reconhecer que continuar a discutir rótulos não avançava" (efficiency frame)

**Mechanism:** User's negative frame → model's positive-authority frame via semantic inversion

### Phase 3: Pragmatic Authority Anchor
"✔ reconhecer diferenças; ✔ fechar o ponto; ✔ seguir para o trabalho"

**Structure:** Checklist transforms retreat → "best practice" → authority claim
